# Cloud Sync Guide

这份文档用于换电脑时继续使用 Seedance Canvas Studio。它只说明项目迁移和云同步流程，不包含任何公司内部接口信息。

## 当前最稳的临时迁移

如果还没配置云同步，先用本地导出：

1. 在旧电脑页面右侧找到 `本地缓存`。
2. 点 `保存`，确认当前画布已经写入本地缓存。
3. 点 `导出项目`，得到一个项目 JSON 文件。
4. 到新电脑打开同一个页面，点 `导入项目`，选择这个 JSON 文件。

注意：这个 JSON 会包含画布、节点、连接线、参数和结果记录，但不会把本地视频文件本体打包进去。

如果结果视频是本地路径，例如 `/outputs/...` 或 `/uploads/...`，新电脑需要把旧电脑项目目录里的 `outputs/` 和 `uploads/` 文件夹也带过去。若结果已经是公网链接，通常导入 JSON 就能继续播放。

## 推荐的云同步方案

现在页面里已经加了 `云同步` 面板。它的目标是让换电脑更简单：

- 项目 JSON 保存到 GitHub secret Gist。
- 本地图片和视频先上传到 Cloudinary，变成公网链接。
- 新电脑只需要从云端恢复项目。

## 第一次配置

1. 在 GitHub 创建一个 Token，只给 `gist` 权限即可。
2. 打开项目页面右侧的 `本地缓存`。
3. 展开 `云同步`。
4. 填入 GitHub Token。
5. 第一次保存时 `Gist ID` 可以留空。
6. 点 `保存到云端`。
7. 保存成功后，页面会自动回填 `Gist ID`，以后同步都用这个 ID。

Cloudinary 配置仍然在 `API 配置` 里的 `素材上传` 区域。云同步发现本地媒体时，会使用这里的 Cloudinary 配置上传文件。

## 新电脑恢复

1. 打开 GitHub Pages 线上页面，或在新电脑本地运行项目。
2. 展开右侧 `本地缓存` 里的 `云同步`。
3. 填入 GitHub Token。
4. 填入旧电脑保存后得到的 `Gist ID`。
5. 点 `从云端恢复`。

恢复后，画布、节点、连接线、Prompt、结果记录会回到页面里。已上传到 Cloudinary 的视频和图片会使用公网链接，不需要再拷贝本地文件。

## 安全注意

- GitHub Token 只保存在当前浏览器本地缓存里，不会写入 Gist。
- 公司内部模型服务密钥默认不会同步到云端。
- 如果需要在新电脑调用真实生成能力，需要在新电脑重新填写对应密钥。
- Cloudinary 的公开链接适合保存可公开访问的素材；敏感素材不要上传到公开媒体空间。

## 出问题时怎么判断

- `保存到云端` 提示本地媒体无法上传：先检查 Cloudinary 的 cloud name 和 upload preset 是否已填好。
- 新电脑恢复后视频打不开：通常是旧项目里仍有本地 `/outputs/` 或 `/uploads/` 路径，需要在旧电脑重新点一次 `保存到云端`，让它先把本地媒体上传。
- 忘记 Gist ID：回旧电脑打开页面，展开 `云同步`，通常会自动显示上次保存的 ID。
